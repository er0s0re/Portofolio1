Jaringan Multi-Gedung

Perjalanan merancang dan mengimplementasikan jaringan fungsional, aman, dan otomatis menggunakan Cisco Packet Tracer.

Mulai Perjalanan Lihat Proyek

Perjalanan Pembelajaran (5 Part)

Desain & Perencanaan

Membangun fondasi dengan topologi Hub-and-Spoke. Memilih perangkat yang tepat (Router vs Switch) dan jenis kabel (Cross vs Straight) untuk setiap koneksi.

Pemberian Alamat IP

Memberi identitas unik ke setiap 'pintu' router. Menggunakan perintah `ip address` dan menyalakan port dengan `no shutdown` untuk membangun konektivitas dasar.

Routing & Pemetaan Jaringan

Menjadi "GPS" bagi paket data. Mengkonfigurasi Static dan Default Route (`ip route`) agar semua jaringan bisa saling berkomunikasi tanpa tersesat.

Wireless & DHCP Server

Menyiapkan akses untuk pengguna. Mengatur SSID & WPA2-PSK di Access Point, serta mengkonfigurasi DHCP Server di router untuk alokasi IP otomatis.

Simulasi & Kontrol Admin

Menguji skenario nyata. Mensimulasikan pengguna pindah gedung dan menjadi "Admin Tiran" dengan memutus koneksi via `shutdown` dan Access Control List (ACL).

Hasil Akhir & Teknologi

Topologi Jaringan Final

Teknologi yang Digunakan

  • Static & Default Routing

    Membangun peta jaringan manual agar semua segmen terhubung.

  • DHCP Server on Router

    Otomatisasi alokasi IP address untuk manajemen klien yang mudah.

  • WLAN & WPA2-PSK

    Menyediakan konektivitas nirkabel yang aman untuk pengguna akhir.

  • Access Control List (ACL)

    Filter lalu lintas untuk kontrol akses dan keamanan tingkat lanjut.

Bukti Kontrol Admin

Tes membuktikan admin dapat memutus koneksi satu gedung penuh dari pusat kendali.

Koneksi Normal: BERHASIL

> ping 10.10.10.1
> Reply from 10.10.10.1: bytes=32 time=2ms TTL=127
> Packets: Sent = 4, Received = 4, Lost = 0 (0% loss)

Setelah Interface di-Shutdown: GAGAL

> ping 10.10.10.1
> Reply from 20.20.20.1: Destination host unreachable.
> Packets: Sent = 4, Received = 0, Lost = 4 (100% loss)
// Blocked by Admin at the Core Router
