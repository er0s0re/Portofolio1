Studi Kasus
Membangun Jaringan Kantor dari Nol

Perjalanan merancang dan mengimplementasikan jaringan fungsional, aman, dan otomatis menggunakan Cisco Packet Tracer.

Mulai Perjalanan Lihat Proyek

Perjalanan Pembelajaran (7 Part)

1

Konektivitas Dasar

Memahami konsep paling dasar: menghubungkan dua PC secara langsung menggunakan kabel Cross-Over dan melakukan tes `ping` untuk pertama kalinya.

2

Membangun LAN dengan Switch

Naik level dengan menggunakan Switch untuk membangun Local Area Network (LAN) dengan topologi Star, menghubungkan banyak PC secara efisien.

3

Segmentasi VLAN

Mulai bermain dengan keamanan dan kerapian jaringan. Memecah satu LAN fisik menjadi dua LAN virtual (VLAN 10: IT, VLAN 20: Marketing) yang terisolasi.

4

Inter-VLAN Routing

Menjadi "mak comblang" jaringan. Menggunakan Router-on-a-Stick untuk menjembatani komunikasi antar VLAN yang berbeda secara terkontrol.

5

DHCP Server Otomatis

Menjadi "pemalas" yang efisien. Mengkonfigurasi Router sebagai DHCP server untuk memberikan alamat IP secara otomatis ke semua PC di setiap VLAN.

6

Keamanan ACL

Menjadi "satpam galak". Menerapkan Access Control List (ACL) untuk memfilter lalu lintas, mengizinkan IT mengakses Marketing, tapi tidak sebaliknya.

7

Koneksi Internet (NAT)

Membuka gerbang ke dunia luar. Mengkonfigurasi NAT agar semua PC dengan IP Private dapat terhubung ke jaringan "internet" simulasi.

Hasil Akhir & Teknologi

Topologi Jaringan Final

Teknologi yang Digunakan

  • Router Cisco 2911 & Switch 2960

    Perangkat inti untuk routing, switching, dan keamanan.

  • VLAN & Inter-VLAN Routing

    Segmentasi jaringan untuk efisiensi dan keamanan.

  • DHCP Server on Router

    Otomatisasi alokasi IP address untuk manajemen mudah.

  • Access Control List (ACL)

    Filter lalu lintas untuk kontrol akses antar VLAN.

  • Network Address Translation (NAT)

    Menghubungkan jaringan lokal ke internet global.

Bukti Keberhasilan ACL

Tes membuktikan bahwa departemen IT bisa mengakses Marketing, namun tidak sebaliknya, sesuai aturan keamanan.

Ping dari IT ke Marketing: BERHASIL

> Reply from 192.168.20.10: bytes=32 time=1ms TTL=128
> Reply from 192.168.20.10: bytes=32 time=2ms TTL=128
> Packets: Sent = 4, Received = 4, Lost = 0 (0% loss)

Ping dari Marketing ke IT: GAGAL

> Request timed out.
> Request timed out.
> Packets: Sent = 4, Received = 0, Lost = 4 (100% loss)
// Blocked by Access Control List
